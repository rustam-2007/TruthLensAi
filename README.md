## 🚀 Key Architectural Features

### 1. Cyber-Forensics Scanning Suite
*   **Dynamic Error Level Analysis (ELA):** Amplifies compression ratio variances across localized matrices, instantly highlighting non-homogeneous boundaries typical of AI generation or spliced composite nodes.
*   **Chroma Channel Isolation:** Segregates chrominance frequencies to expose localized, highly saturated software touch-ups, airbrushing, or generative expansion seams.
*   **Edge Contour Extraction:** Evaluates structural high-frequency noise gradients to audit line boundaries, ensuring pixel alignment and light-direction coherence.

### 2. Multi-Mode Comparative Sandbox
*   **Interactive Split Slider:** Seamlessly wipe between the reconstructed real frame and spliced AI markers using an interactive drag-and-drop boundary line.
*   **Dual Panel (Side-by-Side):** Detailed independent rendering of unprocessed inputs right beside live AI-derived structural heatmaps.
*   **Targeted Inconsistency Spotting:** Overlays dedicated indicators targeting typical generative artifacts: facial asymmetry, pupil reflection mismatch, warped backgrounds, and repeating texture patterns.

### 3. Fail-Safe High-Availability Backend
*   **Auto-Retry Core with Exponential Backoff:** Re-fires network transactions up to three times with backoff strategies to prevent socket failures.
*   **Graceful Overload Fallbacks:** Automatically detects Gemini API congestion limits (such as 503 HTTP statuscodes) and shifts to full client-side mathematical heuristics without interrupting operations.
*   **Zero-Config Demo Mode:** Smart credential checks auto-detect omitted `GEMINI_API_KEY` configurations and substitute high-quality simulation pipelines seamlessly.

## 📦 Local Installation & Deployment

Ensure you have [Node.js (LTS version recommended)](https://nodejs.org/) installed.

### 1. Clone & Enter Directory
```bash
git clone https://github.com/rustam-2007/TruthLensAi.git
cd TruthLensAi
```

### 2. Install Project Dependencies
```bash
npm install
```

### 3. Establish Local Environments
Duplicate `.env.example` to `.env` and fill in your Gemini credentials:
```bash
cp .env.example .env
```
Inside `.env`:
```env
GEMINI_API_KEY=your_actual_gemini_api_key_here
```

### 4. Execute Dev Server
```bash
npm run dev
```
Open [http://localhost:3000](http://localhost:3000).

### 5. Production Compilation
Securely compile components and spin up the production CJS bundle:
```bash
npm run build
npm start
```
